Publish a package

GitHub Pages cannot accept real uploads from visitors by itself. This page gives package authors the exact folder format and metadata needed to publish packages through Git commits, GitHub Actions, R2, S3, Drive, or your own storage API.

1Create your .luma package.
2Upload the file to /storage or external storage.
3Add package metadata to packages/index.json.
4Commit and push to GitHub Pages.

Package metadata example

{
  "id": "aurora-notes",
  "name": "Aurora Notes",
  "version": "1.0.0",
  "file": "../storage/aurora-notes_1.0.0.luma",
  "install": "luma install aurora-notes"
}

Repository setup

Use this repo index URL in LUMA:

https://username.github.io/luma-repo-website/packages/index.json

*“Infinite storage” means the frontend can point to unlimited external storage providers. GitHub Pages itself is not infinite and should only host the website and metadata.
